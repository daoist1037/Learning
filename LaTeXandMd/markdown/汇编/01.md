#### 内存储器
存储速度较快，容量相对较小
+ 内存按单元组织，每个单元都对应一个唯一的地址
+ 每个内存单元存放`1Byte`数据
+ 内存单元的个数称为内存的容量
有关术语：
+ 存放的数据量。用字节表示。每`8`位0/1称为`1Byte`
+ 操作：读和写
  + 读是非破坏性操作，写是破坏性操作
内存储器的分类：（按工作方式）
+ 随机存取存储器(`RAM`)
+ 只读存储器(`ROM`)————今天的可读写的只读存储器，写操作是有条件的，有限的

### 冯诺依曼计算机
工作原理：
存储程序工作原理<br>
结构特点：
运算器为核心<br>
### 工作过程
程序计数器`PC: Programme Counter`<br>
程序计数器存放第一条指令的地址，然后自加（增加的内容取决于上条指令所占据的空间）

读取一条指令的工作过程
1. 将指令所在地址赋给程序计数器`PC`
2. 将`PC`内容送到地址寄存器`AR`，`PC`自动加`1`
3. 把`AR`的内容通过地址总线送至内存存储器，经地址译码器译码，选中相应单元
4. `CPU`的控制器发出读命令
5. 在读命令控制下，把所选择的单元的内容，（即指令操作码）经数据总线送到数据寄存器`DR`
6. 指令译码
  + 数据寄存器`DR`将它送到指令寄存器`IR`，然后再送到指令译码器`ID`
  
传统冯诺依曼特点：
+ 程序存储，共享数据，顺序执行
+ 属于顺序处理机，适合于确定的算法和数值数据的处理
传统冯诺依曼不足：
+ 与存储器间有大量数据交互，对总线要求很高
+ 执行顺序由程序决定，对大型复杂任务较困难
+ 以运算器为核心，处理效率较低
+ 由`PC`控制执行顺序，难以进行真正的并行处理

哈佛结构
+ 指令和数据分别存放在两个独立的存储器模块中
+ `CPU`与存储器间指令和数据的传送分别采用两组独立的总线
+ 可以在一个机器周期内同时获得指令操作码和操作数

### BCD码
+ `Binary Code Decimal` 码
  + 用二进制表示的十进制数
  + `8421BCD`编码
    + 用`4`位二进制码表示`1`位十进制数，每`4`位之间有一个空格
    + 1010————1111是非法BCD码，只是合法的十六进制数
  + BCD码与十进制数之间存在直接对应关系
  + BCD码与二进制的转换，先转换为十进制，再转换为二进制数，反之同样

+ `BCD`码在计算机中的存储方式
  + 以压缩`BCD`码形式存放
    + 一个存储单元中存放2位BCD数
  + 以扩展`BCD`码形式存放    
    + 一个存储单元中存放1位BCD数，高4位为0，低四位为有效位

`ASCII`码
+ 将每个字母、数字、标点、控制符用`1Byte`二进制码表示
+ 标准`ASCII`的有效位；`7bit`，最高位默认为0

`ASCII`码的奇偶校验

## 计算机中的二进制数表示
数的表示方法
+ 定点数（小数点位置固定的数
  + 定点整数————小数点在最右侧
  + 定点小数————小数点在最左侧
  + 不足
    + 编程时需要确定小数点位置
    + 难以表示两个相差比较大的数，（同时呈现一个极大的数和一个极小的数，会需要极长的字节
    + 存储空间利用率低
+ 浮点数（更多采用这种方法）————小数点的位置可以左右移动的数
  + 规格化浮点数
    + 尾数部分用纯小数表示，即小数点右边第1位不为0
  
|   $E_s$    | $E_1,E_2,\dots,E_m$  |  $M_s$   | $M_1,M_2,\dots,M_n$ |
| :--------: | :------------------: | :------: | :-----------------: |
|    阶符    |         阶码         |   数符   |        尾数         |
| 指数的正负 | 指数大小（整数形式） | 数的正负 |     数的有效值      |
数的性质
+ 无符号数
+ 有符号数

无符号和有符号统称为机器数
### 无符号数
算数运算
+ 加法运算
  + 1+1=0 有进位
+ 减法运算
  + 0-1=1 有借位
+ 乘法
  + 每乘以2，相对于被乘数向左移动1位
+ 除法
  + 每除以2，相对于被乘数向右移动1位，最低位消失的数是余数

### 有符号数
+ 最高位表示符号，其余是数值
  + 0表示正，1表示负
+ 原码
  + $[X]_{原}=符号位+|绝对值|$
    + 计算机中用原码进行加减运算比较困难
    + 0 的表示不唯一
+ 反码
  + $若X>0,则[X]_反=[X]_原$
  + $若X<0,则[X]_反 对应原码阿达符号位不变，数值部分按位求反$
    + 数0的表示不唯一
+ 补码
  + $若X>0,则[X]_补=[X]_原$
  + $若X<0,则[X]_补=[X]_反+1$
  + **将减法运算转化为加法运算**
  + 加减乘除变成 **加法和移位** 的运算
    + 以钟表为例：指针从5拨到1
      + 逆时针 5-4=1
      + 顺时针 5+8=12+1=1
      + 对模12，有：
        + 5-4=5+8
        + [-4]补=12-4=8
        + 5-4=5+(-4)=5+(12-4)=5+8=12+1
+ 特殊数 10000000
  + 无符号数 10000000=128
  + 原码 10000000=-0
  + 反码 10000000=-127
  + 补码 10000000=-128
